## Option 3: Configure in eas.json (For Custom Keystore)

If you have the keystore file locally:

### Step 1: Add Keystore to Project
Create a `credentials` folder:
```bash
mkdir credentials
```

Place your `.keystore` or `.jks` file there.

### Step 2: Update eas.json

Add to your production build config:

```json
{
  "build": {
    "production": {
      "android": {
        "resourceClass": "medium",
        "buildType": "app-bundle",
        "credentialsSource": "local"
      }
    }
  }
}
```

### Step 3: Create credentials.json

Create `credentials/android/credentials.json`:

```json
{
  "android": {
    "keystore": {
      "keystorePath": "credentials/your-keystore.jks",
      "keystorePassword": "YOUR_KEYSTORE_PASSWORD",
      "keyAlias": "YOUR_KEY_ALIAS",
      "keyPassword": "YOUR_KEY_PASSWORD"
    }
  }
}
```

## Option 4: Get SHA-1 from Existing EAS Keystore

If EAS already manages your keystore:

```bash
eas credentials -p android
# Select: Keystore: Download
```

Then verify the SHA-1:
```bash
keytool -list -v -keystore path/to/keystore.jks -alias YOUR_ALIAS
```

## üö® Important Notes

### About SHA-1 Fingerprints

- **Development SHA-1**: Used for local testing
- **Production SHA-1**: Used for production builds
- **EAS Build SHA-1**: Generated by EAS (if using EAS credentials)

### For Firebase/Google Services

You typically need to add **multiple SHA-1 fingerprints**:
1. Your development keystore SHA-1
2. Your production keystore SHA-1
3. EAS Build keystore SHA-1 (if using EAS)
